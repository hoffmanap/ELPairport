Texas OD & Catchment Console
Airport origin-destination, catchment, and leakage intelligence

A working analysis tool built from anonymized mobile device mobility data, tracking where airport visitors live, which airports compete for the same travelers, and how much shared traffic exists between airports across the Texas–New Mexico–northern Mexico region.

Catchment

For any tracked airport: where its travelers actually live, county by county, plus state and country breakdowns. Answers "who is this airport actually serving?"

Leakage

Which airport actually wins each county's traffic, where that competes with a neighboring airport, and where people bypass their geographically nearest airport entirely. Answers "who are we losing travelers to, and where?"

Destinations

Shared traveler volume between airports — an interactive connection map showing which airports' traveler populations overlap, and by how much. Answers "where else do our travelers show up?"

How to read this data
  • □ It's a sample, not a census. Every figure is a share of the observed device panel for that period, not a count of actual passengers. Panel size varies quarter to quarter, so raw counts aren't directly comparable across time — use the indexed trend views for that.
  • □ Two different data sources feed the Destinations tab, and they answer different questions. Ping-level trajectory data can show true direction (a device left airport A, later appeared at airport B). The cross-visit device-count matrix — used for most quarters — can only show that a device was seen at both airports sometime in the same period, with no way to know which came first. The tool labels these differently on purpose; "co-visited" never implies a flight sequence.
  • □ "Nearest airport" is pure geography, computed independently of any device data — straight-line distance from a county's center to each airport. When it disagrees with which airport a county's data actually favors, that's a real signal, not a data error.
